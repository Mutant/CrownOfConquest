Changelog

0.7.1 - 12/03/2011

Features

  • New map tiles for the wilderness

Bug Fixes & Minor Additions

  • Improve support for IE8+
  • Items found in treasure chests or on the ground now indicated whether they have an enchantment
  • Buildings now provide garrisons with a 'view' of the surrounding sector, which will be reported on in the Garrison's messages
  • Added an 'auto-cast' option for spell casters, which allows the character to select spells based on the settings in their spell tab

Changelog

0.7.1 - 17/02/2011

Features

  • Added 'Rare' creatures to dungeons
  • Dungeons can now have multiple floors
  • Parties can now build structures in the wilderness, which requires a certain amount of resources that can be bought in towns.
  • Added a way to get more turns, by voting for Kingdoms in games listings.
  • New tilesets for dungeons
  • New character action menus

Bug Fixes & Minor Additions

  • Parties and monsters can no longer flee through dungeon walls
  • Entrances to dungeons that a party has found are now displayed when the party enters a sector, even if they're no longer high enough level to enter it
  • Killing guards in town raids no longer counts towards monster quests
  • Fixed bug that caused characters waiting for recruitment equipment not to be equipped
  • Characters can no longer be removed from town garrison if party is not in town
  • 'Buy' link on character screen now checks party has enough gold, etc. correctly
  • Fixed a bug that caused treasure chests to get filled incorrectly
  • Fixed a bug that caused towns to get free guard hire if they paid for an advisor
  • Adding or removing characters from inn or street no longer screwed up rank separator
  • Added maximum to town sales tax
  • A new quest is no longer automatically created once the last one is taken for the town
  • Fixed a bug that prevented combat from ending correctly if a party is wiped out, but has characters in the inn/street, etc.
  • Monsters now attack parties that are offline in a dungeon.
  • Upgraded items can now be bought and sold at shops
  • Added a 'graffiti' wall to the street of each town.
  • Tweaked Watcher calculations
  • Changed xp required for each character level, and raised the level cap to 25

0.7.0 - 16/01/2011

Features

  • Mayors can now pay for an advisor, who will give advice up to once a day, depending on how much you pay him
  • New enchantment type: Movement Factor bonus
  • Characters can now be garrisoned in towns you control, and will fight alongside the mayor
  • Dungeons now have 'teleporters', which take you to another location in the dungeon
  • New Priest Spell: Portal - lets you leave a dungeon and return to the wilderness immediately
  • Added Town "Inn" and "Street", areas to leave characters in a town

Bug Fixes & Minor Additions

  • Made Garrison name change UI a bit more intuitive
  • Daily email now reports on characters who have no ammo or broken weapons
  • Fixed a bug that meant garrisons with no name had a blank in the town news
  • Fixed a bug that let a character who had been killed during the round be healed later in that round
  • Characters in morgue no longer appear in list of characters in shops (and a few other places)
  • Fixed a bug that let new characters have more than the allowed stats pool by using decimals
  • Fixed a bug that led to multiple entries in "Known Dungeons", and entries not getting updated if a dungeon disappeared
  • The "Status" of a character is now displayed on the charater's screen. This should make it easier to find characters.
  • Election candidates must now remain in town until the election, at the inn.

0.6.9 - 13/12/2010

Features

  • Encumbrance is now "stepped", i.e. a character is slowed down as they reach their encumbrance maximum
  • Added a new enchantment type: "Featherweight"
  • Increased daily turns

Bug Fixes & Minor Additions

  • Characters' action now defaults to 'Attack' rather than 'Defend' after casting a spell or using an item.
  • Fixed a bug that could cause item quests to be incompletable if the dungeon gets deleted
  • Garrison and mayor characters no longer get attacked during party offline combat
  • Added a minimum distance garrisons are allowed to be created from other garrisons
  • Character action menus no longer appear if there are no actions available
  • Closed shops now get removed from towns
  • Parties in a dungeon are no longer displayed in the sector in the wilderness
  • Peasants no longer complain about a town not having an election in a while, if one is already scheduled
  • Killed or overthrown mayors now get added to the cemetery of the town, and must be resurrected.
  • Recent party messages now displayed in the daily email
  • Tweaked calculations for monster generation in the wilderness
  • Changed calculations for repair cost at the blacksmith
  • Elections are cancelled if a town is overtaken before the election occurs
  • Peasants will revolt immediately if tax rate is set too high - preventing tax from being collected
